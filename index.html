Concept 1

Parameters vs Arguments

Although the terms parameter and argument are often used interchangeably, they refer to different concepts in Python function programming.

1. Parameters

A parameter is a variable specified in a function definition. Parameters define the inputs that a function expects to receive when it is called.

Characteristics of Parameters

Declared in the function header
Act as placeholders for incoming values
Exist only within the function scope
Determine the number and type of inputs a function can accept

Example

def calculate_area(length, width):
    return length * width

Here, length and width are parameters.

2. Arguments

An argument is the actual value provided to a function when it is invoked.

Characteristics of Arguments

Supplied during the function call
Assigned to the corresponding parameters
Can be literals, variables, expressions, or objects
Provide the data that the function processes

Example

calculate_area(10, 5)
# 10 is assigned to parameter length
# 5 is assigned to parameter width

In this call, 10 and 5 are arguments.

Concept 2

Types of Arguments in Python

Python supports several types of arguments, allowing functions to be called in flexible ways.

# Type Description
1Positional ArgumentsAssigned based on position in the function call
2Keyword ArgumentsPassed using parameter names, order does not matter
3Default ArgumentsParameter has a preset value when no argument is provided
4Variable-Length Positional (*args)Accepts multiple positional arguments as a tuple
5Variable-Length Keyword (**kwargs)Accepts any number of keyword arguments as a dictionary

1. Positional Arguments

Positional arguments are assigned to parameters based on their position in the function call. The order of arguments must match the order of parameters.

def student(name, age):
    print("Name:", name)
    print("Age:", age)

student("Rahul", 20)

Output

Name: Rahul
Age: 20

2. Keyword Arguments

Keyword arguments are passed using parameter names. Order does not matter, which improves code readability.

def student(name, age):
    print("Name:", name)
    print("Age:", age)

student(age=20, name="Rahul")

Output

Name: Rahul
Age: 20

3. Default Arguments

A parameter can be assigned a default value. If no argument is provided, the default value is used.

def greet(name, message="Welcome"):
    print(message, name)

greet("Rahul")

Output

Welcome Rahul

greet("Rahul", "Good Morning")

Output

Good Morning Rahul
Makes parameters optional
Reduces the number of required arguments

4. Variable-Length Arguments (*args)

When the number of arguments is unknown, Python provides *args to accept multiple positional arguments. Internally, they are stored as a tuple.

def add(*numbers):
    total = sum(numbers)
    print(total)

add(10, 20)
add(10, 20, 30, 40)

Output

30
100
How it works: When you call add(10, 20, 30), Python stores them internally as numbers = (10, 20, 30). *args stores arguments as a tuple.

5. Variable-Length Keyword Arguments (**kwargs)

**kwargs allows a function to accept any number of keyword arguments. Internally, they are stored as a dictionary.

def student_info(**details):
    print(details)

student_info(name="Rahul", age=20, city="Hyderabad")

Output

{'name': 'Rahul', 'age': 20, 'city': 'Hyderabad'}
How it works: The keyword arguments are stored as a dictionary: details = {'name': 'Rahul', 'age': 20, 'city': 'Hyderabad'}
Concept 3

Scope of Variables

Scope refers to the area of a program where a variable can be accessed or used. In simple terms, scope determines where a variable is visible and available in your code.

Every variable in Python has a scope, and Python uses scope rules to decide which variable to use when the same name appears in different places.

Why Understanding Scope Matters
It controls the visibility of variables, prevents naming conflicts, improves code organization and readability, and helps manage memory efficiently.

1. Local Scope

A variable created inside a function is called a local variable. Its scope is limited to that function only. Once the function finishes execution, the local variable is destroyed.

2. Global Scope

A variable defined outside all functions is called a global variable. Global variables can be accessed from anywhere in the program.

To modify a global variable inside a function, use the global keyword.

Defined outside functions
Accessible throughout the program
Remains in memory during program execution
Use global keyword to modify inside a function

3. Enclosing Scope

Enclosing scope occurs in nested functions. A variable defined in the outer function can be accessed by the inner function.

Here, message belongs to outer(). inner() can access it because it is in the enclosing scope.

To modify an enclosing variable, use the nonlocal keyword. It tells Python that the variable belongs to the nearest enclosing function, not the local function.

4. Built-in Scope

Python provides many predefined functions and objects that are available everywhere. These belong to the built-in scope. No need to define or import them.

print()
len()
max()
min()
sum()
type()
Concept 4

The LEGB Rule

Python searches for a variable name in the following order. The first match found is used.

L — Local Variables declared inside the current function
↓
E — Enclosing Variables in the enclosing function (for nested functions)
↓
G — Global Variables declared at the top level of the module
↓
B — Built-in Names in the built-in Python namespace (print, len, etc.)
Key Rule: Python always searches from the innermost scope outward. If a name is found in Local scope, Python does not continue searching further. If no match is found anywhere, a NameError is raised.
